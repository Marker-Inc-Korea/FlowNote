# 탐색기형 메인 화면

## 목적

Windows WPF 메인 화면은 현장/관리자가 폴더 트리에서 문서를 찾고, 문서를 등록하고, 현재 문서의 상태와 버전을 확인하는 작업 화면이다.

## 화면 구성

- 상단: 로그인 사용자, 사용자 관리, 알림, 이력, 작업순서, 보고서, 새 폴더, 문서 등록, 상태 변경, 공개, 파일 감시, 파일 업로드 버튼
- 좌측: 문서 폴더 트리
- 우측 상단: 검색/태그 입력 영역
- 우측 중앙: 문서 목록
- 하단: 현재 작업 상태 메시지

## 기본 폴더

로컬 SQLite 초기화 시 루트 아래 다음 시스템 폴더를 만든다.

- 문서
- 인수인계
- 작업지시서
- 사진

`문서` 아래에는 다음 분류 폴더를 만든다.

- 도면
- 작업표준
- 점검표
- 품질검사
- 안전수칙
- 보전작업
- 일반문서

시스템 폴더는 삭제 대상이 아니다.

## 문서 등록 규칙

- 파일 업로드와 Drag & Drop은 파일을 `Files/Uploads/yyyy-MM-dd/` 아래 복사한다.
- 새 문서는 `WORKING` 상태와 버전 `v1`로 저장한다.
- `문서` 폴더에 등록하면 파일명/문서 유형 기반으로 하위 분류 폴더에 자동 배치한다.
- `인수인계`와 `사진` 폴더는 오늘 날짜 `yyyy-MM-dd` 하위 폴더에 배치한다.
- `작업지시서` 폴더는 파일명에서 확장자를 뺀 값을 작업 제목으로 사용한다.
- 태그 입력값과 폴더/문서 유형/확장자 기반 태그를 함께 저장한다.

## 문서 보기

문서 목록을 더블클릭하면 문서 보기 창을 연다. 현재 미리보기는 텍스트, PDF, Excel 첫 시트, 이미지 파일을 지원한다. 문서 보기 창은 열람 시작/종료 로그를 남기고 설정 시간 이후 자동 닫힘을 수행한다.

## FieldComment

문서 보기 창에서 작성한 새 코멘트는 `field_comments`에 저장되며 문서 버전을 증가시키지 않는다. 첨부 사진/파일은 `field_comment_attachments`와 로컬 `Files/FieldCommentAttachments/` 아래 저장된다.

## 서버 연동

서버 URL과 Bearer token이 있으면 문서, FieldComment, 첨부, 접근 로그를 서버로 전송한다. 실패하면 로컬 저장은 유지되고 `server_sync_queue`에 실패 사유가 남는다.
