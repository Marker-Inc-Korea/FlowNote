# 탐색기형 셸 화면

## 목적

현장 사용자가 처음 체감할 FlowNote 클라이언트 화면은 윈도우 탐색기처럼 문서를 찾고 상태를 확인하는 화면이다. 현재 WPF 구현은 이 흐름을 검증하기 위한 초기 셸이다.

## 현재 화면 구조

- 상단 작업 막대: FlowNote 앱명, 알림함, 새 폴더, 문서 등록, 파일 업로드 버튼 자리
- 좌측 탐색 영역: 루트와 기본 폴더를 표시하는 폴더 트리
- 오른쪽 작업 영역: 검색 입력, 파일 목록, Drag & Drop 파일 업로드
- 하단 상태 영역: 클라이언트 상태 메시지 자리

## 좌측 트리 기본 폴더

초기 실행 시 로컬 SQLite에 루트 폴더 `Root`를 만들고, 그 아래에 다음 네 개 기본 폴더를 만든다.

- `문서`
- `인수인계`
- `작업순서`
- `사진`

이 네 개 폴더는 시스템 폴더로 처리한다. 사용자는 기본 폴더를 삭제할 수 없고, 기존 로컬 DB에 같은 이름의 폴더가 있으면 초기화 과정에서 시스템 폴더로 보정한다.

`문서` 폴더 아래에는 실제 작업 문서를 더 찾기 쉽도록 다음 분류 폴더를 자동 생성한다.

- `도면`
- `작업표준서`
- `점검표`
- `품질검사`
- `안전수칙`
- `보전작업`
- `일반문서`

이 분류 폴더도 시스템 폴더로 처리한다. 기존 DB에서 `문서` 바로 아래에 있던 문서는 초기화 과정에서 파일명과 제목 기준으로 적절한 분류 폴더로 이동한다.

## 현재 코드 위치

- WPF 앱 프로젝트: `apps/windows/src/FlowNote.Windows.App/`
- 코어 모델 프로젝트: `apps/windows/src/FlowNote.Windows.Core/`
- 화면 XAML: `apps/windows/src/FlowNote.Windows.App/MainWindow.xaml`
- 화면 바인딩 모델: `apps/windows/src/FlowNote.Windows.Core/Explorer/`

## 오른쪽 파일 목록과 보기 흐름

왼쪽 폴더 트리 오른쪽의 기본 화면은 파일 목록이다. 문서 내용은 기본 화면에 바로 표시하지 않고, 사용자가 파일 목록에서 항목을 더블 클릭했을 때 별도 보기 창으로 연다.

- 파일 목록에는 파일명, 제목, 유형, 상태, 버전, 사용자, 버전 생성 시간이 표시된다.
- 사용자 열에는 문서 등록자 또는 드롭한 로그인 사용자의 이름을 표시한다.
- 버전 생성 시간 열에는 문서 버전이 생성된 시간을 표시한다.
- 파일을 목록 영역에 끌어 놓을 수 있다.
- 드롭된 파일은 `Data\Files\Uploads\yyyy-MM-dd\` 아래로 복사하고 SQLite 문서로 즉시 저장한다.
- `파일 업로드` 버튼으로 파일 선택 대화상자를 열고 여러 파일을 같은 방식으로 SQLite에 즉시 저장할 수 있다.
- 저장된 문서는 `documents`와 `document_versions`에 원본 버전 `v1`로 기록한다.
- 목록 항목을 더블 클릭하면 파일 보기 창을 연다.
- 로컬 경로가 있는 저장 파일은 지원 형식에 따라 내용을 미리 표시한다.
- PDF 파일은 WebView2 기반 PDF 뷰어로 원본 레이아웃을 표시한다.
- PDF 뷰어 표시가 실패하는 경우에는 PdfPig 기반 텍스트 파싱과 단순 PDF 문자열 추출 방식으로 보정한다.
- Excel 파일은 첫 번째 시트를 텍스트 줄이 아니라 행/열 그리드로 표시한다.
- DB에 메타데이터만 등록된 문서는 현재 클라이언트에서 파일 본문을 사용할 수 없다는 안내를 표시한다.
- 저장된 문서는 파일 보기 창에서 코멘트를 남길 수 있다.
- 새 코멘트를 저장하면 `field_notes`에 현장 코멘트 원천 이력으로 남기고 문서 버전은 증가시키지 않는다.
- 문서 목록용으로 `documents.latest_comment`를 갱신하고 알림함에는 문서 작성자 대상 FieldNote 알림을 남긴다.
- 서버 URL(`FLOWNOTE_API_BASE_URL`)이 설정된 경우에만 로컬 저장 직후 `POST /api/v1/field-notes` 전송을 한 번 시도한다. 서버 URL이 없거나 전송에 실패해도 로컬 저장은 성공으로 유지하고 별도 자동 재시도 큐는 만들지 않는다.
- 과거 호환용 코멘트 버전 증가 경로는 남아 있으며, 이 경로에서는 알림 수신자가 직전 버전 작성자이다.
- 사진 문서는 이미지 미리보기 아래에 누적 코멘트와 새 코멘트 입력 영역을 표시한다.

## 기본 폴더별 등록 규칙

파일 등록, 파일 업로드 버튼, Drag & Drop 업로드는 왼쪽 트리에서 선택한 폴더를 기준으로 기본 배치 규칙을 적용한다.

- `문서`: 파일명과 제목을 기준으로 `도면`, `작업표준서`, `점검표`, `품질검사`, `안전수칙`, `보전작업`, `일반문서` 중 하나에 자동 배치한다.
- `인수인계`: 파일 등록 시 `yyyy-MM-dd` 형식의 날짜 하위 폴더를 만들고 그 아래에 배치한다.
- `작업순서`: 파일명에서 확장자를 제외한 값을 작업 제목으로 자동 생성한다.
- `사진`: 파일 등록 시 `yyyy-MM-dd` 형식의 날짜 하위 폴더를 만들고 그 아래에 배치한다.
- 루트가 선택되었거나 선택 폴더가 없으면 기본 대상은 `문서` 폴더이다.

## 현재 범위

포함된 것:

- WPF 앱 프로젝트 생성
- 코어 프로젝트 생성
- 탐색기형 레이아웃 구성
- 로컬 SQLite 폴더와 문서 목록 바인딩
- 루트 아래 기본 폴더 4개 자동 생성
- `문서` 아래 작업 문서 분류 폴더 자동 생성
- 기본 폴더 삭제 차단
- 기본 폴더별 문서 등록 규칙
- 오른쪽 파일 목록 기본 화면
- 파일 목록 Drag & Drop 업로드와 SQLite 즉시 저장
- 파일 업로드 버튼을 통한 SQLite 즉시 저장
- 파일 목록 더블 클릭 보기 창
- PDF 파일 원본 레이아웃 보기
- Excel 파일 행/열 그리드 보기
- 파일 보기 창 FieldNote 저장과 최신 코멘트 요약 갱신
- FieldNote 알림함
- 과거 호환용 코멘트 버전 증가와 직전 버전 작성자 대상 알림 경로
- 로그인 성공 후 탐색기 화면 표시

포함하지 않은 것:

- 운영용 FastAPI 서버 자동 동기화
- 업로드 확정 API 호출
- 파일 감시
- 다운로드 차단
- 문서 뷰어 자동 닫힘
