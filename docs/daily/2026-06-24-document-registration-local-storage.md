# 2026-06-24 문서 등록과 로컬 저장소 기록

이 문서는 문서 등록과 파일 저장 작업을 현재 코드 기준으로 정리한 기록이다.

## 서버 문서 등록

- `POST /api/v1/documents`는 문서 메타데이터와 파일을 함께 등록한다.
- 서버는 파일을 로컬 `storage/` 하위에 저장한다.
- 문서 버전에는 파일명, 저장 경로, MIME 타입, 확장자, 파일 크기, SHA-256 해시를 기록한다.
- 문서 상태는 등록 시 `WORKING`, `IN_REVIEW`, `ARCHIVED` 중 하나를 사용할 수 있다.
- 공개 문서는 업로드만으로 자동 확정되지 않으며, 별도 공개 흐름을 거친다.

## Windows 로컬 저장

- Windows 앱은 공통 SQLite `data/local/flownote.local.sqlite`를 사용한다.
- 문서 등록 시 로컬 파일을 앱 관리 저장소로 복사하고 문서/버전 메타데이터를 DB에 기록한다.
- 신규 문서는 기본적으로 `WORKING` 상태와 `v1` 버전으로 시작한다.
- 문서 파일 자체는 Git 커밋 대상이 아니라 로컬 보존 대상이다.

## 보존 규칙

테스트 등록 파일, 출력 파일, 로그, SQLite DB는 이후 검증 근거이므로 사용자가 명시적으로 삭제를 지시하지 않는 한 삭제하지 않는다.
