# FlowNote MVP 범위

## 현재 MVP 구현

현재 코드 기준으로 MVP에 포함되어 실제 구현된 범위는 다음과 같다.

- Windows WPF 로그인과 탐색기형 문서 화면
- 로컬 SQLite 기반 문서/폴더/사용자/이력 저장
- FastAPI 서버 로그인과 Bearer token 인증
- 문서 등록, 버전 추가, 상태 변경, 공개 버전 지정
- 문서 태그
- 문서 미리보기, 열람 로그, 자동 닫힘, 다운로드 차단
- FieldComment 원천 기록과 첨부
- 알림과 활동 이력
- 사용자 추가/수정/역할 변경/비밀번호 변경
- 작업순서 보드, 항목, 순서 변경, 상태 변경, 이력, 알림 후보, TV 화면
- 보고서 초안 생성 보조와 문서 저장
- 관리자 파일 감시 후보와 버전 확정
- WPF 로컬 저장 후 서버 전송 후보와 서버 ID 매핑

## MVP 판단 기준

MVP의 성공 기준은 AI가 답변하는 것이 아니라 현장 문서와 현장 기록이 지속적으로 쌓이는 것이다. 현장 사용자가 문서를 열람하고, 짧은 FieldComment와 사진/파일을 남기고, 관리자가 이를 보고서나 문서 이력으로 정리할 수 있어야 한다.

## 현재 제외 범위

- AI 검색/작업 조언
- MES/ERP 자동 수신 어댑터
- 운영용 설치 프로그램과 서비스 자동 등록
- 클라우드 배포
- 일반 브라우저 사용자 화면
- 개인 메신저 수집
- GPS 추적
- 근태 관리
- 개인 휴대폰 기본 배포
- CAD 원본 직접 뷰어와 HWP 고급 미리보기

## 후속 MVP 후보

- 서버-WPF 동기화 정책 고도화
- 보고서 서버 동기화
- 관리자 세션/계정 운영 UI
- 운영 백업/복구 절차
- 실제 현장 권한 정책과 계정 발급 절차
