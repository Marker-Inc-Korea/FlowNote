# FlowNote MVP 범위

## 0. 현재 코드 기준 상태

현재 구현 완료로 볼 수 있는 범위는 Windows WPF 로컬 SQLite 프로토타입과 FastAPI SQLite MVP API의 일부이다.

- 로그인 선행 흐름과 기본 계정 `admin / 1234`
- 관리자 그룹과 반장 3명 기준 작업조 개발/스모크 테스트 계정, 샘플/테스트 산출물 보존
- 탐색기형 폴더 트리와 파일 목록
- 삭제 불가 기본 폴더 `문서`, `인수인계`, `작업순서`, `사진`
- `문서` 하위 작업 성격별 분류 폴더 자동 생성
- 파일 업로드 버튼과 Drag & Drop 기반 로컬 파일 복사
- SQLite 문서 등록과 원본 버전 `v1` 저장
- `FLOWNOTE_API_BASE_URL` 설정 시 WPF 로컬 파일 등록 성공 후 같은 파일을 FastAPI `POST /api/v1/documents`로 등록 후보 시도. 서버 등록 실패 시 로컬 등록 유지
- `인수인계`와 `사진`의 날짜 하위 폴더 배치
- `작업순서` 파일명 기반 제목 자동 생성
- TXT, PDF, XLSX, 이미지 미리보기
- 문서 보기 창 열림/닫힘 기준 로컬 SQLite `document_view_logs` 열람 감사 로그
- 문서 보기 창 개발 검증용 기본 타이머 자동 닫힘과 `auto_closed` 닫힘 사유 기록
- 문서 보기 창의 누적 코멘트 최신순 표시와 `field_notes` 기반 코멘트 저장. 신규 WPF 코멘트는 문서 버전을 증가시키지 않음
- 직전 버전 작성자 대상 알림 생성과 알림함
- Windows 앱 빌드와 콘솔형 스모크 테스트
- FastAPI 헬스체크와 DB 헬스체크
- FastAPI 개발용 기본 관리자 계정 생성
- FastAPI 로그인 API: 사용자명/비밀번호 검증, 사용자 정보 반환, Bearer access token, refresh token, 만료 시각 반환
- FastAPI refresh/logout API: refresh token 회전, 현재 세션 폐기, 이전 token 재사용 거부
- FastAPI `/api/v1/auth/me` 현재 사용자 확인 API
- FastAPI 문서/FieldNote/문서 접근 로그 API의 Bearer access token 인증. 인증 정보가 없거나 유효하지 않으면 `401` 반환
- `FLOWNOTE_API_BASE_URL` 설정 시 Windows WPF 로그인 화면의 FastAPI 로그인 API 우선 호출과 로컬 SQLite 로그인 폴백
- FastAPI 문서 등록/목록/상세/버전 API와 서버 로컬 `storage/` 파일 저장
- FastAPI 현장 코멘트 최소 등록/조회/관리자 갱신 API
- FastAPI FieldNote 사진/파일 첨부 등록/조회 API
- FastAPI 문서 접근 로그 등록/조회 API
- FastAPI 문서 상태 전환, 버전 상태 전환, 명시적 공개 버전 지정, 공개 문서 조회 API
- FastAPI 태그 조회/등록, 문서 태그 교체 API
- FastAPI role 기반 권한 검사: 문서 쓰기, 태그 쓰기, FieldNote 등록, 접근 로그 조회
- FastAPI 작업순서판 생성, 항목 추가, 전체 재정렬, 상태 변경, 이력 조회 API
- `FLOWNOTE_API_BASE_URL` 설정 시 Windows 스모크 테스트의 서버 로그인 API, 인증 헤더가 붙은 `/auth/me`, 서버 FieldNote 등록, 서버 문서 접근 로그 등록/조회 검증
- `FLOWNOTE_API_BASE_URL` 설정 시 Windows 스모크 테스트의 서버 FieldNote 첨부와 작업순서판 API 검증

현재 구현되지 않았거나 운영 안정화가 남은 범위는 관리자 강제 세션 폐기 화면, 로컬 열람 로그의 운영용 서버 동기화 정책 고도화, 운영 설정 기반 뷰어 자동 닫힘 값 관리, 고도화된 작업내역 모델, 작업순서판 알림 발송 정책, 보고서, AI 검색/조언, MES/ERP 연동, 다운로드 차단이다. 아래 MVP 범위는 제품 목표와 다음 구현 기준이며 현재 코드 구현 완료 목록이 아니다.

현재 프로그램 테스트와 관련 작업 기준은 다음과 같다.

- 문서 등록 테스트는 여러 테스트 ID로 진행한다.
- 문서 등록 그룹은 조장 이상으로 본다. 조원 계정은 문서 등록자가 아니라 코멘트 작성자로 검증한다.
- 코멘트 등록은 ID 제한을 두지 않는다.
- 프로그램 테스트용 문서 파일은 한글, 숫자, 영문으로 된 파일명과 표시 문구만 사용한다. 실제 화면에서 `???` 같은 깨진 문자가 보이는 파일은 테스트 기준에 맞지 않는다. 이 제한은 프로그램 테스트용 문서 파일에만 적용하며, 일반 문서 파일이나 MD 문서 작성 기준에는 적용하지 않는다.
- 서버 URL인 `FLOWNOTE_API_BASE_URL`이 없으면 로컬 SQLite 기준으로 검증을 진행한다. 서버 URL이 설정된 경우에만 FastAPI 연동 검증을 추가한다.

## 1. MVP 목표

1차 MVP는 Python FastAPI 서버와 Windows WPF 클라이언트로 문서 등록, 버전 관리, 권한 기반 열람, 문서 상태, 태그 기반 연결, 기본 현장지식 수집을 검증하는 것이다.

MVP의 목적은 단순히 문서를 올리고 보는 기능에 머무르지 않는다. 현장 공개 문서, 고객 정의 문서 구조, 현장 코멘트, 작업 문제점, 사진 기록, 작업순서가 서로 연결되는 최소 구조를 만들어 추후 AI 검색과 작업 조언에 사용할 원천 데이터를 쌓을 수 있는지 확인한다. 완성형 DMS/KMS보다 현장의 소리를 문서와 함께 기록하는 운영 흐름을 먼저 검증한다.

현장 사용자가 처음 체감해야 하는 기능은 AI가 아니라 익숙한 탐색기형 검색, 보기 전용 뷰어, 사진 업로드, 작업순서 확인이다. 따라서 1차 MVP에는 파일명, 문서명, 태그, 문서 구조, 작업지시 기준의 기본 검색과 관리자 입력 기반 작업순서판을 포함한다.

MVP의 성공 기준은 AI 기능 완성이 아니라 현장 사용자가 실제로 문서를 열고, 짧은 코멘트와 사진 기록을 남기며, 메신저나 구두로 흩어지던 내용을 FlowNote 기록으로 전환할 수 있는지 확인하는 것이다.

현장 코멘트는 원천 이력으로 저장한다. 1차 MVP에서는 코멘트와 작업자/작업그룹, 문서, 태그의 연결을 남기는 데 집중하고, 관리자 분석 보고서와 AI 보조 보고서 초안은 후속 단계에서 구현한다.

MVP는 MES/ERP를 대체하지 않는다. 작업지시는 관리자가 직접 입력하는 구조를 기본으로 한다. 기존 MES/ERP가 있으면 외부 참조 ID와 매핑 구조를 준비하되, 실제 현장별 연동 어댑터와 자동 수신은 후속 단계에서 확장한다.

문서 구조는 고객이 결정한다. BOM 문서 구조라는 표현은 현장에서 문서를 계층적으로 정리해 부르는 예시일 뿐이며, MES BOM을 차용하거나 1차 MVP의 기본 구조로 강제하지 않는다.

개발은 Python FastAPI 서버와 Windows WPF 클라이언트를 기준으로 진행한다. 고객 생산공장 운영에서는 일반 브라우저 직접 접근을 기본 사용 방식으로 두지 않는다.

배포 방식은 사내 서버형을 기준으로 한다. 1차 MVP는 서버 PC 1대에 FastAPI 서버, SQLite DB, 로컬 storage 폴더를 두고 클라이언트 PC에 설치파일을 배포하는 구조를 전제로 API와 설정을 설계한다.

MVP 개발은 기능을 빠르게 구현하고 실제 사용 여부를 검증하는 방식으로 진행한다. 기술 검증을 별도 선행 단계로 오래 두기보다 FastAPI, SQLite, 로컬 storage, Windows WPF 클라이언트를 먼저 적용해 기능을 만들고, 구현 중 확인되는 한계와 보완점을 다음 단계의 판단 근거로 삼는다.

현장의 이야기는 최대한 반영하되, 현재 개발 언어, 클라이언트 방향, 사내 서버형 배포, 보안 정책과 맞지 않는 요청은 검토 후 결정한다. 기존 시스템과의 상세 연동은 후속 단계에서 맞추며, 1차 MVP에서는 외부 참조 ID와 기본 매핑 구조만 준비한다.

현장 의견은 `수용`, `불수용`, `검토`로 분리한다. 사진 기록, 짧은 코멘트, 인수인계, 작업순서 확인, 문서 보기 개선은 MVP 검증에 반영하되, 메신저 전체 기능, 개인 메신저 수집, GPS 추적, 근태 관리, 전체 MES/ERP 대체 기능은 MVP 범위에서 제외한다.

현장 관찰은 MVP 요구사항 분석의 일부이다. 작업자의 감각 판단, 공정 중간 보류, 재작업, 선별 이유처럼 MES 결과에 남지 않는 맥락을 짧은 코멘트, 사진, 태그, 작업내역으로 남길 수 있는지 확인한다.

장갑 낀 손, 터치 장비, 별도 현장 단말기 같은 사용 형태는 초기부터 단정하지 않는다. 먼저 현장에서 요청한 흐름을 단순한 UI로 구현해 실제 사용 여부를 확인하고, 장비나 입력 방식의 제약이 검증되면 후속 단계에서 조정한다. 별도 장비 투입은 기능 검증 목적의 실험으로 보며, 구매나 상시 운영은 공장과 비용 투입 여부를 회의한 뒤 결정한다.

단말기는 원하는 위치에 있다고 가정하지 않는다. MVP 검증에서는 단말기 위치, OS, 성능, 통신 품질, 작업자 동선 안에서 실제 입력 가능한 순간을 함께 확인한다.

- 회원 로그인
- 역할 기반 권한
- 관리자/현장 사용자 구분
- 문서 메타데이터 저장
- 파일 저장소 저장
- 문서 등록
- 고객 정의 문서 구조 생성
- 관리자 입력 기반 작업지시서 구조 생성
- 문서 구조 항목에 문서 연결
- 문서 버전 업로드
- 변경 사유 필수 기록
- 최근 등록 버전과 현장 공개 버전 조회

- 문서 또는 문서 구조 항목에 연결된 기본 현장 코멘트 등록
- 현장 코멘트 원문 저장
- 현장 코멘트 또는 작업일지성 기록의 사진 첨부 저장
- 작업일지나 종이 기록을 사진으로 남기는 흐름 지원
- 현장 코멘트 입력자, 실제 전달자 또는 작업자/작업그룹 기록
- 신호등 방식 상태 기록
- 기본 정형 문구 선택
- 관리자 대리 입력
- 파일명, 문서명, 태그, 문서 구조, 작업지시 기준 기본 검색
- 관리자 입력 기반 작업순서판
- 작업순서 항목 상태 관리: 대기, 진행중, 완료, 보류
- 작업순서 변경 이력과 알림 이벤트 생성
- 외부 시스템 참조 ID 저장
- 기본 연동 매핑 구조
- 배포 형태와 외부 접근 정책 설정

### 보안

- 로그인하지 않은 사용자 접근 차단
- 역할 기반 화면 접근 제어
- 다운로드 차단은 Windows WPF 클라이언트 앱에서 구현
- 뷰어 자동 닫힘 기본 타이머는 Windows WPF 클라이언트 앱에서 구현하고 로컬 열람 로그로 기록

## 3. 1차 MVP 제외

- 고도화된 Windows 파일 감시
- 현장 코멘트 관리자 검토와 정리 문구 관리
- 관리자 분석 내용 관리
- 정형 문구 템플릿 고도화
- 보고서 생성
- AI 보조 보고서 초안 생성
- AI 검색과 작업 조언. 단, 파일명/문서명/태그 기반 기본 검색은 1차 MVP에 포함
- MES 연동 기반 작업자 코멘트 수집
- 현장별 MES/ERP 어댑터 고도화
- 메신저 전체 기능
- 개인 메신저 수집 또는 스크래핑
- GPS 추적
- 근태 관리
- 개인 휴대폰 APK 또는 iOS 앱을 기본 배포 경로로 삼는 것
- 과거 생산 데이터 기반 신규 작업 자동 설계
- 자동 의사결정
- HWP/DWG 고급 미리보기
- CAD 원본 직접 뷰어와 CAD-PDF 자동 변환
- 대용량 multipart upload
- 기존 문서 대장 또는 작업지시서 파일 파싱

## 4. 1차 MVP 성공 기준

- 관리자가 문서를 등록할 수 있다.
- 관리자가 고객이 정한 방식으로 문서 구조를 만들 수 있다.
- 관리자가 문서 구조 항목에 문서를 연결할 수 있다.
- 관리자가 새 버전을 변경 사유와 함께 업로드할 수 있다.
- 관리자가 문서 상태를 작업중, 검토중, 공개, 보관 등으로 구분할 수 있다.
- 관리자가 문서와 현장 코멘트에 설비, 품목, 공정, 오류 유형 같은 태그를 달 수 있다.
- 현장 코멘트에 입력자와 실제 전달자 또는 작업자/작업그룹을 남길 수 있다.
- 현장 사용자가 Windows WPF 클라이언트에서 현장 공개 문서를 열람할 수 있다.
- 현장 사용자가 탐색기형 목록에서 문서명, 파일명, 태그, 구조 기준으로 문서를 찾을 수 있다.
- 현장 사용자가 현재 문서 또는 문서 구조 항목에 짧은 코멘트를 등록할 수 있다.
- 현장 사용자가 사진을 첨부한 현장 기록을 남길 수 있다.
- 현장 사용자가 신호등 방식 또는 기본 정형 문구로 부담 없이 기록을 남길 수 있다.
- 현장 사용자가 작업일지나 종이 기록을 사진으로 남길 수 있다.
- 관리자가 현장에서 전달받은 내용을 대신 입력할 수 있다.
- 관리자가 작업순서를 입력하고 조정할 수 있다.
- 현장 사용자가 TV용 화면에서 현재 작업순서를 확인할 수 있다.
- 문서 메타데이터는 SQLite에 저장된다.
- 파일 바이너리는 서버 로컬 storage 폴더에 저장된다.
- 작업지시서는 관리자 입력으로 생성할 수 있다.
- 기존 MES/ERP를 대체하지 않고 추후 외부 참조 ID를 통해 연결할 수 있다.
- 현장 사용자가 직관적인 UI에서 별도 설명 없이 핵심 기능을 찾고 사용할 수 있다.
- 문서 열람, 기본 검색, 짧은 현장 코멘트, 사진 기록, 작업순서 확인 중 어떤 기능이 실제로 사용되는지 확인할 수 있다.
- 작업자가 실제 공정 중 멈추거나 기록할 수 있는 입력 지점을 확인할 수 있다.
- 공정 중간 판단, 보류, 재작업, 선별 이유를 짧은 코멘트나 사진 기록으로 남길 수 있다.
- 현장 단말기의 위치, OS, 사양, 통신 상태가 핵심 화면 사용에 문제 없는지 확인할 수 있다.
- 별도 장비나 특수 입력 장치가 없어도 기본 사용 흐름을 검증할 수 있다.
- 별도 장비가 필요한 경우 기능 검증 목적과 공장 비용 투입 결정 사항을 구분해 기록할 수 있다.

## 5. 2차 이후 전제

1차에서 만든 Python FastAPI는 Windows WPF 클라이언트에서 사용한다.
따라서 1차 구현 시 사내망 앱 환경에서 안정적으로 동작할 수 있도록 인증, 파일 표시, 권한 UI, 로컬 기능 호출 범위를 설계한다.
