# FlowNote MVP 범위

## 1. MVP 목표

1차 MVP는 Web UI와 API만으로 문서 등록, 버전 관리, 권한 기반 열람, 문서 상태, 태그 기반 연결, 기본 현장지식 수집을 검증하는 것이다.

MVP의 목적은 단순히 문서를 올리고 보는 기능에 머무르지 않는다. 현장 공개 문서, 고객 정의 문서 구조, 현장 코멘트, 작업 문제점이 서로 연결되는 최소 구조를 만들어 추후 AI 검색과 작업 조언에 사용할 원천 데이터를 쌓을 수 있는지 확인한다. 완성형 DMS/KMS보다 현장의 소리를 문서와 함께 기록하는 운영 흐름을 먼저 검증한다.

현장 코멘트는 원천 이력으로 저장한다. 1차 MVP에서는 코멘트와 작업자/작업그룹, 문서, 태그의 연결을 남기는 데 집중하고, 관리자 분석 보고서와 AI 보조 보고서 초안은 후속 단계에서 구현한다.

MVP는 MES/ERP를 대체하지 않는다. 작업지시는 관리자가 직접 입력하는 구조를 기본으로 한다. 기존 MES/ERP가 있으면 외부 참조 ID와 매핑 구조를 준비하되, 실제 현장별 연동 어댑터와 자동 수신은 후속 단계에서 확장한다.

문서 구조는 고객이 결정한다. BOM 문서 구조라는 표현은 현장에서 문서를 계층적으로 정리해 부르는 예시일 뿐이며, MES BOM을 차용하거나 1차 MVP의 기본 구조로 강제하지 않는다.

개발은 Web UI와 API를 기준으로 진행하지만, 고객 생산공장 운영에서는 일반 브라우저 직접 접근을 기본 사용 방식으로 두지 않는다. 1차 MVP의 Web UI는 이후 Android/Windows 클라이언트 WebView에 탑재될 기준 화면이다.

Android와 Windows 앱은 1차 MVP 이후 단계에서 WebView 기반으로 붙인다.

배포 방식은 고객 데이터 주권을 기준으로 한다. 1차 MVP는 특정 설치 위치를 강제하지 않고, 내부망 로컬 서버, 고객 사내망 서버, 인가된 외부 접근 서버, 고객 승인 전용 클라우드에 배포 가능한 구조를 전제로 API와 설정을 설계한다.

## 2. 1차 MVP 포함

### 서버/API

- MySQL 연결
- 회원 로그인
- 역할 기반 권한
- 관리자/현장 사용자 구분
- 문서 메타데이터 저장
- 파일 저장소 저장
- 문서 등록
- 고객 정의 문서 구조 생성
- 관리자 입력 기반 작업지시서 구조 생성
- 문서 구조 항목에 문서 연결
- 문서 버전 업로드
- 변경 사유 필수 기록
- 최근 등록 버전과 현장 공개 버전 조회
- 문서 상태 관리: 작업중, 검토중, 공개, 보관
- 문서 태그 등록: 설비, 품목, 공정 등
- 현장 코멘트 태그 등록: 설비, 품목, 공정, 오류 유형 등
- 문서 접근 로그
- 문서 또는 문서 구조 항목에 연결된 기본 현장 코멘트 등록
- 현장 코멘트 원문 저장
- 현장 코멘트 입력자, 실제 전달자 또는 작업자/작업그룹 기록
- 신호등 방식 상태 기록
- 기본 정형 문구 선택
- 관리자 대리 입력
- 외부 시스템 참조 ID 저장
- 기본 연동 매핑 구조
- 배포 형태와 외부 접근 정책 설정

### Web UI

- TypeScript + React + Vite
- 로그인 화면
- 문서 목록
- 문서 상세
- 문서 등록
- 문서 구조 화면
- 문서 구조 항목 생성/수정
- 구조 항목별 문서 연결
- 새 버전 업로드
- 변경 사유 입력
- 문서 상태 선택
- 문서 태그 입력
- 현장 공개 문서 뷰어
- 현재 문서 또는 문서 구조 항목에 대한 짧은 현장 코멘트 입력
- 현장 코멘트 태그 입력
- 정상/주의/문제 같은 신호등식 기록
- 기본 정형 문구 선택
- 1차 Web UI에서는 다운로드 차단과 뷰어 자동 닫힘을 구현하지 않음

### 보안

- 로그인하지 않은 사용자 접근 차단
- 역할 기반 화면 접근 제어
- 다운로드 차단은 Android/Windows 클라이언트 앱 단계에서 구현
- 뷰어 자동 닫힘은 Android/Windows 클라이언트 앱 단계에서 구현

## 3. 1차 MVP 제외

- Android Kotlin + WebView 앱
- Windows WPF + WebView2 앱
- Windows 파일 감시
- WebView 브릿지 실제 구현
- 현장 사용자 다운로드 차단 실제 구현
- 뷰어 자동 닫힘 실제 구현
- 현장 코멘트 관리자 검토와 정리 문구 관리
- 관리자 분석 내용 관리
- 정형 문구 템플릿 고도화
- 보고서 생성
- AI 보조 보고서 초안 생성
- AI 검색과 작업 조언
- MES 연동 기반 작업자 코멘트 수집
- 현장별 MES/ERP 어댑터 고도화
- 과거 생산 데이터 기반 신규 작업 자동 설계
- 자동 의사결정
- HWP/DWG 고급 미리보기
- 대용량 multipart upload
- 기존 문서 대장 또는 작업지시서 파일 파싱

## 4. 1차 MVP 성공 기준

- 관리자가 문서를 등록할 수 있다.
- 관리자가 고객이 정한 방식으로 문서 구조를 만들 수 있다.
- 관리자가 문서 구조 항목에 문서를 연결할 수 있다.
- 관리자가 새 버전을 변경 사유와 함께 업로드할 수 있다.
- 관리자가 문서 상태를 작업중, 검토중, 공개, 보관 등으로 구분할 수 있다.
- 관리자가 문서와 현장 코멘트에 설비, 품목, 공정, 오류 유형 같은 태그를 달 수 있다.
- 현장 코멘트에 입력자와 실제 전달자 또는 작업자/작업그룹을 남길 수 있다.
- 현장 사용자가 WebView에 탑재될 기준 Web UI에서 현장 공개 문서를 열람할 수 있다.
- 현장 사용자가 현재 문서 또는 문서 구조 항목에 짧은 코멘트를 등록할 수 있다.
- 현장 사용자가 신호등 방식 또는 기본 정형 문구로 부담 없이 기록을 남길 수 있다.
- 관리자가 현장에서 전달받은 내용을 대신 입력할 수 있다.
- 문서 메타데이터는 MySQL에 저장된다.
- 파일 바이너리는 파일 저장소에 저장된다.
- 작업지시서는 관리자 입력으로 생성할 수 있다.
- 기존 MES/ERP를 대체하지 않고 추후 외부 참조 ID를 통해 연결할 수 있다.

## 5. 2차 이후 전제

1차에서 만든 Web UI와 API는 Android와 Windows 앱에서 그대로 재사용한다.  
따라서 1차 구현 시 WebView 환경에서도 동작할 수 있도록 라우팅, 인증, 파일 표시, 권한 UI를 설계한다.
