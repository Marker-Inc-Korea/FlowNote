# 0011. 문서관리와 현장지식관리의 균형

## 상태

Accepted

## 배경

FlowNote는 생산현장의 기술문서, 도면, 매뉴얼, 작업지시서를 관리해야 한다. 동시에 현장 사용자가 남기는 짧은 코멘트, 작업 평가, 문제점, 개선 의견도 문서와 연결해 축적해야 한다.

이때 제품 방향이 단순 문서관리 프로그램으로만 좁아지면 현장 경험 데이터가 쌓이지 않는다. 반대로 순수 지식관리 시스템으로 확장되면 현장 공개 문서, 버전, 권한, 열람 보안이라는 핵심 요구가 흐려질 수 있다.

## 결정

FlowNote는 문서관리와 현장지식관리를 융합하되 어느 한쪽으로 치우치지 않는다. 초기에는 완성형 DMS나 KMS가 아니라, 오래 운영하면서 현장 경험과 문서 이력을 함께 축적하는 기반 시스템으로 둔다.

- 문서관리 영역은 파일 저장, 메타데이터, 버전, 변경 사유, 권한, 열람 보안, 이력을 책임진다.
- 현장지식 영역은 현장 코멘트, 작업 평가, 문제점, 관리자 정리 문구, 보고서 원천 데이터를 책임진다.
- 관리자 분석과 보고서화는 현장 코멘트를 실제 활용 가능한 정제 데이터로 바꾸는 중간 계층이다.
- 작업 맥락 영역은 고객 정의 문서 구조, 작업지시서, 작업내역 버전을 문서와 현장지식에 연결한다.
- AI 기능은 위 데이터를 검색하고 작업 조언으로 활용하는 계층이다. 초기에는 검색, 근거 제시, 작업 전 주의 사항 제공을 중요 기능으로 두고, 자동 의사결정이나 신규 작업 자동 설계는 장기 가능성으로만 둔다.
- 현장 사용자의 경험은 관리 대상 데이터이다. 짧은 코멘트라도 문서와 작업 맥락에 연결해 남김으로써 특정 개인에게만 머물던 노하우를 조직 지식으로 전환한다.
- 현장 입력은 처음부터 많은 텍스트를 강제하지 않는다. 신호등식 기록, 정형 문구, 짧은 메모, 관리자 대리 입력을 통해 단계적으로 정착시킨다.

1차 MVP에는 최소 현장지식 수집과 작업자/작업그룹 추적을 포함한다. 단, 관리자 분석, 정형 문구 템플릿 고도화, 보고서 생성, AI 검색과 조언은 후속 단계로 분리한다.

## 영향

- `Document`, `DocumentVersion`, `FileObject`는 문서관리 데이터로 유지한다.
- `FieldNote`는 문서 또는 문서 버전, 문서 구조 항목, 작업내역에 연결되는 현장지식 데이터로 관리한다.
- `Report`와 `ReportSource`는 원천 코멘트와 정제 보고서 사이의 추적성을 유지한다.
- `WorkRecord`와 `WorkRecordVersion`은 작업 맥락 데이터로 분리한다.
- AI 인덱스는 원본 데이터가 아니라 참조 데이터로만 사용한다.
- 위키, 게시판, 자유 지식베이스 같은 기능은 초기 범위에서 제외한다.
- 충분한 운영 데이터가 쌓이기 전에는 AI가 판단을 대신하지 않으며, 검색과 조언 중심으로 제한한다.
