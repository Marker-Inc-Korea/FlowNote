# 0002. 생산공장 문서 버전 관리와 단말기 알림

## 상태

Accepted

## 결정

동일 문서는 `Document`로 유지하고, 파일 개정은 `DocumentVersion`으로 누적한다.

새 버전 등록 시 다음 정보를 기록한다.

- 버전 번호
- 표시용 버전명
- 변경 사유
- 등록자
- 등록일
- 파일 정보

FlowNote는 최근 등록 버전과 현장 공개 버전을 분리해서 관리한다. 현장 단말기는 기본적으로 현장 공개 버전을 열람한다.

문서 버전이 올라가면 `DOCUMENT_VERSION_PUBLISHED` 알림 이벤트를 생성한다.

## 권한 기준

- 현장 사용자: 문서 열람, 코멘트 등록
- 관리자: 문서 등록, 수정, 삭제, 새 버전 업로드, 권한 관리

## 근거

파일 덮어쓰기는 변경 이력과 현장 전파 추적을 어렵게 만든다. 버전과 변경 사유를 강제하면 어떤 문서가 언제, 왜 바뀌었는지 추적할 수 있다.

## 결과

- `DocumentVersion.change_reason`은 필수이다.
- 현장 문서 열람은 `published_version_id`를 기준으로 한다.
- 버전 상승 알림은 단말기 알림의 핵심 이벤트이다.
