# 0013. MES/ERP 보완 연동 전략

## 상태

Accepted

## 배경

많은 생산공장은 이미 스마트팩토리 도입 과정에서 MES를 구축했거나 ERP를 MES처럼 사용하고 있다. FlowNote가 이 시스템들을 대체한다는 전제를 두면 현장 도입 장벽이 커지고 제품 범위도 불필요하게 커진다.

MES/ERP는 작업지시, 품목, 공정, 설비, 생산실적 같은 정형 데이터를 잘 다룬다. 하지만 정형 기계 데이터와 관리자 보고서만으로 AI를 구성하면 현장작업자의 노하우, 실제 문제점, 말로 전달되는 개선 의견이 빠질 수 있다.

## 결정

FlowNote는 MES/ERP를 대체하지 않는다.

- 현재 작업지시는 관리자가 직접 입력하는 구조를 기본으로 한다.
- 기존 MES/ERP가 있으면 후속 단계에서 현장별 API 어댑터로 데이터를 받아들인다.
- 기존 시스템의 원본 데이터는 해당 시스템에 두고, FlowNote는 외부 참조 ID와 매핑을 관리한다.
- FlowNote는 문서, 현장 코멘트, 작업 문제점, 관리자 보고서, 작업내역을 연결해 AI 활용 데이터를 보강한다.
- 라인, 공정, 설비, 품목 개념은 생산현장 맥락을 표현하는 기준 후보로 차용하되, 고객별 용어와 문서 구조에 맞게 유연하게 매핑한다.
- 기존 MES/ERP가 없는 현장에서는 관리자 입력 기반의 최소 업무 데이터 수집 구조를 사용한다.
- 제품 목표는 특정 업무 시스템 대체가 아니라 데이터 수집과 활용이다.

## 영향

- `ExternalSystem`과 `IntegrationMapping` 모델을 둔다.
- 문서 구조와 작업내역은 외부 작업지시, 품목, 공정 등 참조 ID를 가질 수 있다.
- AI 검색과 조언은 MES/ERP 정형 데이터와 FlowNote 현장지식 데이터를 함께 참조한다.
- 현장별 연동 방식이 다르므로 통합 코어와 현장별 어댑터를 분리한다.
- MVP에서는 관리자 입력 기반 작업지시 구조와 기본 참조 필드, 매핑 구조를 준비하고, 현장별 어댑터 구현과 자동 수신은 후속 단계로 확장한다.
