# 0015. 태그, 문서 상태, 클라이언트 보안 범위

## 상태

Accepted

## 배경

FlowNote는 단순히 문서를 올리고 코멘트를 등록하는 시스템이 아니다. 고객 정의 문서 구조에 모든 관계를 담을 수 없으므로 설비, 품목, 공정, 오류 유형 같은 태그가 필요하다. 또한 업로드된 문서가 항상 최신 확정본이라고 가정하면 작업중 문서와 현장 공개 문서를 구분할 수 없다.

다운로드 차단과 뷰어 자동 닫힘은 중요 보안 기능이지만, 실제 운영에서는 Android/Windows 클라이언트 앱이 제어해야 한다. 1차 Web/API에서 이 기능을 완성하려고 하면 실제 운영 방식과 맞지 않는다.

## 결정

- BOM 문서 구조는 현장 용어의 예시일 뿐이며, MES BOM 차용이나 기본 구조 강제를 뜻하지 않는다.
- 문서와 현장 코멘트는 설비, 품목, 공정, 오류 유형 등 태그를 가질 수 있어야 한다.
- 문서 구조로 표현되지 못한 관계는 태그로 보완한다.
- 문서는 작업중, 검토중, 현장 공개, 보관 같은 상태를 가진다.
- 최근 등록 버전과 현장 공개 버전은 분리한다.
- 1차 Web/API MVP에서는 다운로드 차단과 뷰어 자동 닫힘을 실제 구현하지 않는다.
- 다운로드 차단과 뷰어 자동 닫힘은 Android/Windows 클라이언트 앱 단계에서 구현한다.
- 기존 문서 대장 또는 작업지시서 파일 파싱은 최종 목표 중 하나로 두되, 개발 단계는 추후 결정한다.

## 영향

- `TagDefinition`, `DocumentTag`, `FieldNoteTag` 모델을 둔다.
- `Document.status`, `Document.latest_version_id`, `Document.published_version_id`를 구분한다.
- `DocumentVersion.version_status`, `is_latest`, `is_published`를 구분한다.
- 1차 MVP 범위에서 다운로드 차단과 뷰어 자동 닫힘을 제외한다.
- 장기 과제로 기존 문서 대장/작업지시서 파싱 서비스를 고려한다.
