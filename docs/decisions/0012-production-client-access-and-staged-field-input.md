# 0012. 운영 클라이언트 접근과 단계적 현장 입력

## 상태

Accepted

## 배경

FlowNote는 Web UI와 API를 먼저 개발한다. 하지만 생산공장 운영 환경에서는 보안상 일반 사용자의 웹 브라우저 직접 접근을 허용하지 않을 수 있다. 하이브리드 앱 구조는 공장 사용자가 웹으로 접속하게 하기 위한 결정이 아니라, UI를 빠르게 유지보수하고 Android/Windows 화면을 공통으로 관리하기 위한 결정이다.

또한 현장 사용자가 실시간으로 많은 내용을 입력하는 것은 작업 흐름에 직접 들어가야 하므로 초기 안착이 어렵다. 현장에서는 아직 말로 전달하거나 관리자가 메모를 정리하는 방식도 함께 존재한다.

## 결정

- 개발은 Web UI와 API를 기준으로 진행한다.
- 고객 생산공장 운영에서는 승인된 Android/Windows 클라이언트 앱의 WebView 접근을 기본으로 한다.
- 일반 브라우저 직접 접근은 운영 설정과 네트워크 정책으로 차단할 수 있어야 한다.
- 현장 입력은 신호등식 기록, 기본 정형 문구, 짧은 메모, 관리자 대리 입력으로 시작한다.
- MES 연동이나 자동화 시스템 기반 입력은 후속 단계로 둔다.
- 관리자급 사용자는 현장 작업자에게 입력의 필요성과 당위성을 제공하고, 단계적으로 업무 스타일이 바뀌도록 운영을 유도한다.

## 영향

- `FieldNote.input_mode`는 `signal`, `free_text`, `template`, `template_with_text`, `admin_proxy`, `mes_integration`을 고려한다.
- `FieldNote.reported_by`와 `entry_source`를 두어 실제 전달자와 입력자를 구분한다.
- MVP에는 신호등식 기록, 기본 정형 문구 선택, 관리자 대리 입력을 포함한다.
- 고도화 단계에서 MES 연동, 자동 수집, 작업 흐름 내 직접 입력을 확장한다.
