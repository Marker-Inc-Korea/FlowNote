# 0016. 현장 코멘트 분석, 보고서화, 작업자 추적

## 상태

Accepted

## 배경

현장 코멘트는 생산 중 발생하는 중요한 원천 이력이지만, 코멘트만 쌓는다고 곧바로 활용 가능한 데이터가 되지는 않는다. 짧은 문구, 신호등식 기록, 관리자 대리 입력은 현장 안착에는 유리하지만, 실제 의사결정이나 후속 개선에 쓰려면 관리자급 사용자의 분석과 정리가 필요하다.

반대로 관리자 보고서만 남기면 어떤 현장 상황, 어떤 문서, 어떤 작업자 또는 작업그룹의 경험에서 결론이 나왔는지 추적하기 어렵다. FlowNote는 난잡한 원천 이력과 정제된 보고서가 함께 존재해야 과거와 현재, 미래 판단의 근거가 된다고 본다.

## 결정

- 현장 코멘트는 원천 이력으로 보존한다.
- 현장 코멘트는 관리자 검토, 정리 문구, 분석 내용을 가질 수 있어야 한다.
- 보고서는 현장 코멘트, 작업내역, 관련 문서, 태그를 근거로 생성한다.
- AI는 보고서 작성을 대체하지 않고, 관리자급 사용자의 보고서 초안 작성을 보조한다.
- 최종 보고서는 `Document`로 저장하고, 원천 코멘트와 작업내역은 `ReportSource`로 추적한다.
- 문서 수정자, 문서 열람자, 코멘트 입력자, 실제 전달자 또는 작업자/작업그룹을 기록한다.
- 작업자 개인을 항상 식별할 수 없는 현장을 고려해 작업반, 조장, 관리자 대리 등록자 같은 작업 주체도 표현한다.
- 라인, 공정, 설비, 품목 개념은 생산현장 맥락을 표현하기 위해 차용하되 고객별 구조와 용어에 맞게 유연하게 매핑한다.

## 영향

- `OperatorProfile` 모델을 둔다.
- `FieldNote.analysis_content`, `operator_id`, `analyzed_by`, `analyzed_at`을 고려한다.
- `Report`와 `ReportSource`를 상세 모델로 관리한다.
- `WorkRecordParticipant`로 작업내역과 작업자/작업그룹을 연결한다.
- 1차 MVP는 원천 코멘트와 작업자/작업그룹 추적을 포함하고, 관리자 분석 보고서와 AI 보조 보고서 초안은 후속 단계로 둔다.
