# 0008. 회원 인증과 클라이언트 단계 보안 정책

## 상태

Accepted

## 결정

FlowNote는 회원관리 시스템을 기준으로 로그인 기반 인증을 사용한다.

사용자는 로그인 후 역할과 권한에 따라 기능을 사용할 수 있다.

- 현장 사용자: 문서 열람, 코멘트 등록
- 관리자: 문서 등록, 개정, 삭제, 다운로드, 권한 관리
- 시스템 관리자: 회원, 설정, 보안 정책 관리

문서 파일 다운로드는 관리자급 권한에서만 허용하는 것을 목표로 한다. 단, 다운로드 차단의 실제 구현은 Windows WPF 클라이언트 앱 단계에서 처리한다.

문서 뷰어 자동 닫힘도 클라이언트 앱 단계에서 구현한다. 제한 시간은 운영 설정에서 관리한다.

## 근거

생산현장에서는 사용자가 자리를 비운 상태에서 문서가 계속 화면에 노출되는 현실 보안 문제가 발생할 수 있다. 또한 문서 파일이 외부로 유출되면 안 되므로 일반 현장 사용자에게 다운로드 기능을 제공하지 않는다.

회원 로그인과 역할 기반 권한을 전제로 해야 관리자와 현장 사용자의 기능을 명확히 분리할 수 있다.

## 결과

- `UserAccount`, `Role`, `UserRole`, `LoginSession` 모델을 둔다.
- `ViewerSession` 모델로 문서 열람 만료 시간을 관리한다.
- 현장 사용자 권한은 `view`, `comment` 중심으로 제한한다.
- `download` 권한은 관리자급 역할에만 부여한다.
- 다운로드 차단과 뷰어 자동 닫힘은 Windows WPF 클라이언트 앱에서 구현하고 Python FastAPI 서버의 접근 로그 또는 감사 로그와 연결한다.
