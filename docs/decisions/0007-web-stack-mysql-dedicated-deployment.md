# 0007. 웹 스택, MySQL, 고객 데이터 주권 기반 배포

## 상태

Accepted

## 결정

FlowNote의 웹 UI는 TypeScript + React + Vite를 기준으로 작성한다.

메타데이터 데이터베이스는 MySQL을 기본으로 한다.

서비스 배포는 고객 데이터 주권을 위해 고객 또는 생산현장 단위의 독립 인스턴스 구조를 기본으로 한다. 여러 고객이나 현장이 하나의 중앙 서비스를 공유하는 멀티테넌트 구조는 기본값으로 두지 않는다.

독립 인스턴스는 내부망 로컬 서버만을 의미하지 않는다. 내부망 서버, 고객 사내망 서버, 인가된 외부 접근 서버, 전용 호스팅, 고객 승인 전용 클라우드 중 어떤 형태로 운영할지는 고객 보안 정책과 협의해 결정한다.

## 근거

TypeScript + React + Vite는 WebView 기반 업무 UI를 빠르게 개발하고 유지보수하기에 적합하다. Android WebView와 Windows WebView2에서 같은 웹 UI를 사용할 수 있으며, 개발 환경에서는 일반 브라우저로도 검증할 수 있다. 단, 고객 생산공장 운영 환경에서는 일반 브라우저 직접 접근을 기본 사용 방식으로 두지 않는다.

MySQL은 사용자가 익숙한 데이터베이스이며, 문서 메타데이터, 버전, 권한, 이력, 현장 코멘트, 작업내역 같은 관계형 데이터를 관리하기에 적합하다.

생산공장은 보안 요구가 높고, 기술문서와 도면, 생산규격, 작업 보고서를 외부 클라우드에 두는 것을 꺼리는 경우가 많다. 고객 또는 현장별 독립 인스턴스를 기본으로 하면 데이터, 파일 저장소, 인증, 네트워크 정책을 고객 단위로 분리할 수 있고, 데이터 저장 위치에 대한 통제권을 고객에게 둘 수 있다.

## 결과

- `apps/web`은 TypeScript + React + Vite 기준으로 설계한다.
- Android와 Windows 앱은 같은 웹 UI를 WebView로 표시한다.
- 운영 환경의 현장 사용자는 승인된 클라이언트 앱의 WebView를 통해 접근한다.
- MySQL은 FlowNote 메타데이터 DB의 기본 선택지이다.
- 파일 바이너리는 MySQL에 직접 저장하지 않고 파일 저장소에 둔다.
- 배포 설계는 고객 또는 현장별 독립 인스턴스를 기본으로 한다.
- 실제 설치 위치와 외부 접근 허용 여부는 고객 보안 정책과 협의해 결정한다.
- 내부망 전용 운영과 인가된 외부 접근 운영을 모두 고려한다.
