# 0017. 고객 데이터 주권 기반 배포

## 상태

Accepted

## 배경

FlowNote가 다루는 기술문서, 도면, 생산규격, 작업 보고서, 현장 코멘트는 생산현장의 핵심 자산이다. 많은 고객은 이러한 문서를 외부 클라우드에 저장하는 것을 꺼리며, 데이터와 파일 저장 위치를 직접 통제하기를 원한다.

초기에 로컬 서버를 고려한 이유는 보안을 가볍게 보기 때문이 아니라, 고객이 데이터 주권을 유지할 수 있게 하기 위해서이다. 다만 고객마다 보안 정책, IT 인프라, 외부 접근 필요성, 운영 인력이 다르므로 배포 위치를 하나로 정형화할 수 없다.

## 결정

- FlowNote는 고객 또는 현장 단위 독립 인스턴스를 기본으로 한다.
- 데이터베이스, 파일 저장소, 인증, 네트워크 정책은 고객 또는 현장 단위로 분리한다.
- 내부망 로컬 서버를 지원한다.
- 초기 배포는 고객 현장 또는 사내 서버 PC 1대를 기준으로 한다.
- 외부 접근이 필요한 경우 VPN, 방화벽, IP 제한, 인증 정책, 감사 로그를 전제로 한다.
- 실제 설치 위치와 운영 방식은 고객 보안 정책과 협의해 결정한다.
- 클라우드나 외부 접근은 초기 기본값이 아니며, 현장 테스트 이후 필요가 확인되면 별도 확장으로 검토한다.

## 영향

- 배포 문서는 서버 PC 1대와 클라이언트 설치파일 배포를 기본으로 설명한다.
- 운영 설정에는 배포 형태, 외부 접근 정책, 승인 앱/단말기 정책, VPN/방화벽 정책을 포함한다.
- AI 검색과 인덱스도 고객 승인 환경 안에서 동작하는 것을 기준으로 설계한다.
- 고객 데이터와 파일이 승인되지 않은 외부 저장소로 임의 이전되지 않도록 한다.
