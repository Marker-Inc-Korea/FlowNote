# 0010. 고객 정의 문서 구조와 외부 작업 구조 연결

## 상태

Accepted

## 결정

FlowNote는 문서를 단순 목록으로만 관리하지 않고, 고객이 이미 사용하는 문서 정리 구조에 맞게 구성할 수 있게 한다.

문서 구조는 다음 모델로 분리한다.

- `DocumentStructure`: 고객 정의 문서 구조 루트
- `DocumentStructureItem`: 문서 구조 항목
- `DocumentStructureItemDocument`: 구조 항목과 문서 연결

구조 유형은 `custom`, `work_order`, `project`, `folder`, `hierarchy` 등을 사용할 수 있다. 단, 프로그램이 특정 구조를 기본값으로 강제하지 않는다.

사용자는 직접 문서 구조와 항목을 만들고 문서를 연결할 수 있다. 작업지시서 구조도 현재 단계에서는 MES 연동 데이터가 아니라 관리자가 직접 입력한 구조로 관리한다. 추후 MES/ERP 연동 시 외부 프로젝트, 작업지시서, 품목, 공정 데이터를 고객 정의 구조에 매핑할 수 있게 한다.

## 근거

생산현장의 문서는 고객마다 다른 기준으로 정리된다. 어떤 현장에서는 이를 BOM 문서라고 부르기도 하지만, 이는 MES BOM을 차용한다는 뜻이 아니라 현장 문서 정리 방식을 표현하는 용어이다. 프로그램이 구조를 강제하면 고객의 기존 정리 방식을 바꾸게 되므로 도입성이 떨어진다.

문서 구조를 문서와 분리하면 같은 문서를 여러 고객 정의 구조에서 재사용할 수 있고, 특정 문서 버전을 구조 항목에 고정 연결할 수도 있다.

## 결과

- `DocumentLink`는 단순 외부 대상 연결에 사용한다.
- 고객 정의 문서 구조는 `DocumentStructure` 계열 모델로 관리한다.
- 문서 구조 API를 추가한다.
- 1차 Web + API 구현 범위에 고객 정의 문서 구조 생성과 문서 연결을 포함한다.
