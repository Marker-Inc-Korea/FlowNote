# 0014. 관리자 입력 기반 작업지시 우선

## 상태

Accepted

## 배경

FlowNote는 작업지시서 또는 작업지시 기준의 문서 구조를 가질 수 있다. 그러나 현재 단계의 작업지시는 MES와 실시간 연동되는 데이터가 아니다. 생산현장마다 MES 구축 수준이 다르고, ERP를 MES처럼 사용하는 경우도 있으며, 아직 연동 API가 준비되지 않은 현장도 있다.

## 결정

초기 작업지시는 관리자가 직접 입력하는 구조를 기본으로 한다.

- 관리자는 작업지시서 구조를 생성하고 문서를 연결한다.
- 작업지시 번호, 제목, 관련 문서, 작업 메모는 FlowNote 내부 데이터로 관리한다.
- `source_type`은 `manual`을 기본값으로 둔다.
- MES/ERP 연동이 추가되면 `source_type=external`과 외부 참조 ID를 사용해 자동 수신 데이터와 연결한다.
- 후속 연동은 기존 수동 입력 구조를 대체하는 것이 아니라 자동 생성/갱신 경로를 추가하는 방식으로 구현한다.

## 영향

- MVP에서 작업지시서 구조 생성은 관리자 입력 화면을 기준으로 한다.
- MES/ERP 자동 수신 API는 후속 단계로 둔다.
- 데이터 모델은 처음부터 `manual`과 `external` 출처를 구분한다.
- 외부 연동이 추가되어도 기존 수동 작업지시 데이터와 문서 연결은 유지되어야 한다.
