# FlowNote 구현 로드맵

## 1. 기본 전략

FlowNote는 Python FastAPI 서버와 WPF/Avalonia 네이티브 클라이언트를 기준으로 다시 구현한다. 독립 Web UI는 신규 개발하지 않는다.

구현 방향은 문서관리 기능만 먼저 만들고 현장지식은 나중에 붙이는 방식으로 보지 않는다. 1차부터 문서, 고객 정의 문서 구조, 현장 코멘트가 연결되는 최소 구조를 넣어 향후 AI 검색과 작업 조언의 근거 데이터가 자연스럽게 쌓이도록 한다.

현장 체감 우선순위는 AI보다 탐색기형 검색, 보기 전용 뷰어, 사진 기록, 작업순서판과 현장 TV 화면이다. 사용자는 윈도우 탐색기처럼 찾고, 바로 보고, 필요한 사진 기록을 남기고, 현재 처리 순서를 큰 화면에서 확인할 수 있어야 한다.

FlowNote는 MES/ERP를 대체하는 방향으로 개발하지 않는다. 초기 작업지시는 관리자가 직접 입력하는 구조로 시작한다. 기존 MES/ERP가 있으면 후속 단계에서 작업지시, 품목, 공정, 설비, 생산실적 같은 정형 데이터를 자동 수신하도록 확장한다.

과거 생산 기반으로 새로운 작업을 설계하거나 자동 의사결정을 지원하는 기능은 장기 가능성으로 둔다. 현재 구축의 중심은 현장의 소리를 기록하고, 관련 문서와 과거 경험을 검색하며, 작업 전 주의 사항을 조언하는 것이다.

WPF 또는 Avalonia 클라이언트는 실제 운영 프론트엔드이다. 현장 사용자 열람과 코멘트 입력, 관리자 화면, 파일 감시 업로드 보조를 같은 설치형 클라이언트에서 권한에 따라 제공한다. 로컬 제어, 파일 감시, OS 연동은 클라이언트의 네이티브 기능으로 처리한다.

고객 생산공장 운영에서는 일반 브라우저 직접 접근을 기본 사용 방식으로 두지 않는다. 실제 현장 사용자는 승인된 WPF/Avalonia 클라이언트 앱을 통해 접근한다.

현장 입력은 처음부터 많은 직접 입력을 요구하지 않는다. 신호등식 기록, 기본 정형 문구, 짧은 메모, 관리자 대리 입력으로 시작하고, 현장 업무 방식이 바뀐 뒤 MES 연동과 자동화 입력으로 확장한다.

현장 코멘트는 원천 이력으로 시작하지만, 장기 활용을 위해서는 관리자급 분석과 보고서 문서화가 필요하다. 따라서 초기에는 원천 데이터와 추적성을 확보하고, 후속 단계에서 관리자 분석, AI 보조 보고서 초안, 최종 보고서 문서 저장을 확장한다.

배포는 사내 서버형을 기준으로 한다. FlowNote는 서버 PC 1대에 FastAPI 서버, SQLite DB, 로컬 storage 폴더를 두고 클라이언트 설치파일을 배포하는 방향으로 설계한다.

## 2. 단계별 구현 순서

### 1차: FastAPI + 네이티브 클라이언트 MVP

목표는 Python FastAPI 서버와 WPF/Avalonia 클라이언트로 문서 관리의 핵심 흐름과 기본 현장지식 수집 흐름을 동작시키는 것이다.

- Python FastAPI 서버
- SQLite 기반 메타데이터 저장
- 서버 로컬 storage 파일 저장
- 회원 로그인
- 역할 기반 권한
- 관리자/현장 사용자 구분
- 문서 등록
- 고객 정의 문서 구조 생성
- 관리자 입력 기반 작업지시서 구조 생성
- 문서 구조 항목에 문서 연결
- 문서 버전 업로드
- 변경 사유 기록
- 문서 상태 관리
- 문서와 현장 코멘트 태그 관리
- 파일 저장소 저장
- 현장 공개 문서 열람
- 탐색기형 기본 검색
- 기본 현장 코멘트 등록
- 사진 첨부 기반 현장 기록
- 코멘트 입력자와 실제 전달자 또는 작업자/작업그룹 기록
- 신호등식 상태 기록
- 기본 정형 문구 선택
- 관리자 대리 입력
- 문서 또는 문서 구조 항목과 코멘트 연결
- 관리자 입력 기반 작업순서판
- 현장 TV용 작업순서 화면
- 작업순서 변경 이력과 알림 이벤트
- 외부 시스템 참조 ID 저장
- 기본 연동 매핑 구조
- 배포 형태와 외부 접근 정책 설정
- API: `/api/v1`
- 독립 Web UI 제외

### 2차: 클라이언트 현장 기능 고도화

목표는 현장 사용자 단말기 경험을 제공하는 것이다.

- WPF/Avalonia 앱 UI
- 로그인 세션 연동
- 현장 공개 문서 열람
- 문서 변경 알림 확인
- 사진 첨부 기반 현장 기록
- 작업순서 확인
- 현장 코멘트 등록
- 정형 문구 선택
- 신호등식 기록
- 뷰어 자동 닫힘 적용
- 파일 다운로드 차단 유지

### 3차: 관리자 기능과 파일 감시 고도화

목표는 관리자 단말기와 파일 감시 업로드 보조를 제공하는 것이다.

- WPF/Avalonia 관리자 UI
- 관리자 로그인
- 지정 파일 또는 폴더 감시
- 변경 후보 생성
- 새 버전 업로드 보조
- 변경 사유 입력 후 업로드 확정
- 관리자 다운로드와 로컬 파일 제어 정책 적용

### 4차: 현장 코멘트 고도화와 보고서

- 관리자 코멘트 검토
- 원문/정리 문구 분리
- 관리자 분석 내용 작성
- 원천 코멘트와 작업내역을 보고서 근거로 연결
- 정형 문구 템플릿 관리
- 문서별 문제점 조회
- AI 보조 보고서 초안 생성
- 관리자 검토 후 보고서 확정
- 보고서 결과를 문서로 저장

### 4.5차: MES/ERP 연동 고도화

- 현장별 MES/ERP API 어댑터 구현
- 작업지시, 품목, 공정, 설비, 생산실적 수신
- 관리자 입력 작업지시와 외부 수신 작업지시 매핑
- 외부 작업지시와 작업순서판 항목 매핑
- 외부 원본 ID와 FlowNote 문서 구조/작업내역 매핑
- 기존 시스템이 없는 현장의 최소 수집 화면 구성
- 연동 실패와 동기화 이력 관리

### 5차: 검색과 AI

- 본문 추출 기반 검색 인덱스 구성
- 문서/버전/코멘트/보고서/작업내역/외부 정형 데이터 검색
- 작업지시 문서 위험 요소 미리보기
- 작업 조언 생성
- AI 응답 근거 표시
- 보고서 초안 작성 보조 고도화

### 장기 검토: 생산 데이터 기반 의사결정 조언

- 축적된 생산보고서와 작업내역 기반 패턴 분석
- 과거 유사 작업 비교
- 신규 작업 설계 참고 자료 추천
- 의사결정 조언
- 기존 문서 대장, 작업지시서, 관리 파일 파싱 지원

이 단계는 충분한 현장 데이터가 쌓인 뒤 검토하며, 현재 MVP나 1차 AI 범위로 보지 않는다.

## 3. 착수 원칙

- 1차에서는 독립 Web UI를 만들지 않는다.
- 1차에서는 FastAPI 계약과 WPF/Avalonia 앱의 핵심 화면 흐름을 함께 안정화한다.
- 다운로드 차단과 뷰어 자동 닫힘은 클라이언트 앱과 서버 감사 로그를 함께 사용해 구현한다.
- 1차에서는 최소 현장 코멘트 수집과 작업자/작업그룹 추적을 포함하되, 관리자 분석/정형 문구 템플릿 고도화/보고서는 후속 단계로 분리한다.
- 1차에서는 기본 검색과 작업순서판을 포함하되, 본문 추출 기반 고급 검색과 AI 검색은 후속 단계로 분리한다.
- CAD 원본 직접 뷰어와 CAD-PDF 자동 변환은 후속 단계로 두고, 초기에는 PDF 변환본 또는 지원 가능한 미리보기 계층을 우선한다.
- WPF/Avalonia 클라이언트는 앱 UI를 기준으로 구현한다.
- 로컬 기능은 앱에서 필요한 범위가 명확해진 뒤 네이티브 기능으로 구현한다.
- AI와 보고서는 문서/권한/열람 보안이 안정화된 뒤 구현한다.
- 현장 입력은 업무 흐름을 방해하지 않는 수준에서 시작하고, 관리자 주도 운영과 교육으로 단계적으로 정착시킨다.
